Installation

Requirements

  • PHP 8.4 or higher
  • Laravel 12 or higher
  • FFI extension enabled
  • GD extension for image processing
  • FFmpeg (for video encoding/decoding)
  • yt-dlp (for YouTube uploads)

Install Package

composer require shamimstack/youtube-cloud-storage

Publish Configuration

php artisan vendor:publish --provider="Shamimstack\YouTubeCloudStorage\YTStorageServiceProvider" --tag=config

⚠️ Note

The package requires system binaries (FFmpeg, yt-dlp). Run the setup wizard for automatic configuration.

Quick Setup

Interactive Setup Wizard

The easiest way to configure the package is using the interactive setup:

php artisan yt:setup

This will:

  1. Auto-detect system binaries (FFmpeg, FFprobe, yt-dlp)
  2. Compile Wirehair from source if needed
  3. Validate all dependencies via health check
  4. Prompt for YouTube OAuth credentials
  5. Write configuration to your .env file

Manual Configuration

Add these variables to your .env file:

FFMPEG_PATH=/usr/bin/ffmpeg
FFPROBE_PATH=/usr/bin/ffprobe
YTDLP_PATH=/usr/bin/yt-dlp
YOUTUBE_API_KEY=your_api_key_here
YOUTUBE_CLIENT_ID=your_client_id_here
YOUTUBE_CLIENT_SECRET=your_client_secret_here
YOUTUBE_REFRESH_TOKEN=your_refresh_token_here

✅ Setup Complete!

You're now ready to upload files to YouTube!

Usage

Via Storage Facade

use Illuminate\Support\Facades\Storage;

// Upload a file
$disk = Storage::disk('youtube');
$disk->put('backup.zip', file_get_contents('/path/to/file.zip'));

// Download a file
$data = $disk->get('backup.zip');
file_put_contents('/tmp/restored.zip', $data);

// List files
$files = $disk->allFiles();

// Delete a file
$disk->delete('backup.zip');

Via Artisan Commands

Upload Files

# Basic upload
php artisan yt:store /path/to/file.pdf

# With custom redundancy
php artisan yt:store /path/to/file.zip --redundancy=2.0

# With metadata
php artisan yt:store image.png \
  --title="My Image" \
  --description="Uploaded via YTStorage" \
  --privacy=private

Download Files

# Restore from YouTube URL
php artisan yt:restore "https://youtube.com/watch?v=dQw4w9WgXcQ"

# Specify output path
php artisan yt:restore "URL" --output=/tmp/restored.pdf

Programmatic Access

use Shamimstack\YouTubeCloudStorage\Engines\EncoderEngine;

Architecture

How It Works

1

Original File

Your document, image, or archive file

↓
2

Wirehair Fountain Encoder

Splits into N packets + M redundant packets. Any N + 2% can recover original.

↓
3

DCT Sign-Bit Embedding

Convert packets to bitstream, generate video frames, embed bits in DCT coefficient signs (positive=0, negative=1)

↓
4

FFmpeg Video Encoding

RGB24 raw frames → H.264/FFV1 video, uploaded to YouTube as private video

Mathematical Foundation

Discrete Cosine Transform

Forward DCT:  F = T · B · T^T
Inverse DCT:  B = T^T · F · T

Where:
- B = 8×8 pixel block
- F = 8×8 frequency coefficients
- T = DCT basis matrix (orthonormal)

API Reference

AutoConfigurator

Auto-detects system binaries and compiles Wirehair.

$result = $autoConfig->runAutoConfig();

if ($result['allFound']) {
    echo "FFmpeg: {$result['ffmpeg']}\n";
    echo "Wirehair: {$result['wirehair']}\n";
}

HealthCheck

Validates all runtime prerequisites.

$healthCheck = app(HealthCheck::class);
$report = $healthCheck->run();

if (!$report['passed']) {
    foreach ($report['checks'] as $check) {
        if (!$check['passed']) {
            echo "Failed: {$check['name']}\n";
        }
    }
}

EncoderEngine

Pipeline coordinator for encode/upload and download/decode.

$engine = app(EncoderEngine::class);

// Upload
$result = $engine->encodeAndUpload(
    filePath: '/path/to/file.pdf',
    title: 'Document',
    privacyStatus: 'private',
);

// Download
$decodedPath = $engine->downloadAndDecode(
    videoUrl: 'https://youtube.com/watch?v=...',
    outputPath: '/tmp/output.pdf',
);

  • Request quota increase from Google

🔬 Technical Deep Dive: DCT Steganography

Discrete Cosine Transform

Forward DCT:  F = T · B · T^T
Inverse DCT:  B = T^T · F · T

Where:
- B = 8×8 pixel block
- F = 8×8 frequency coefficients
- T = DCT basis matrix (orthonormal)

Sign-Bit Embedding Process

  • • Bit 0: Make coefficient positive (sign = +)
  • • Bit 1: Make coefficient negative (sign = -)
  • • Threshold: Only embed in coefficients with magnitude > threshold
  • • Robustness: Survives H.264 compression and transcoding

📦 Fountain Codes

Wirehair creates systematic and repair packets using O(N) erasure coding.

  • ✓ Any N + 2% packets recover original
  • ✓ Configurable redundancy (default: 1.5x)
  • ✓ CRC32 integrity verification
  • ✓ Resistant to packet loss

💡 Key Innovation

The combination of Wirehair fountain codes (for redundancy) and DCT steganography (for invisibility) creates a robust storage system that survives YouTube's aggressive video compression while remaining completely invisible to viewers.
